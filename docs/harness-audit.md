# Harness 组件假设登记表

每个 harness 组件都编码了一个假设：「没有这个约束，模型会出错」。
随着模型进化，某些假设会失效，对应组件可以简化或移除。

## 审查触发条件

- Claude 或 Codex 发布新的大版本
- 某个约束被模型反复忽略（约束本身可能已成为噪声）
- 某条规则在日常使用中从未触发（可能模型已内化）

## 审查方法

1. 识别：这个组件假设模型不能自主完成什么？
2. 实验：在 3 个真实任务中临时移除该组件，观察输出质量
3. 结论：保留 / 简化（降低约束强度）/ 移除（模型已内化）
4. 更新本表记录决策

## 操作说明

**判断一个约束是否可以移除的标准：**
- 在最近 10 次真实任务中，这个约束触发了几次？
- 去掉它之后，输出质量是否明显下降？
- 是否存在更轻量的替代方案（如从禁止改为警告）？
