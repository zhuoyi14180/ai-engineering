# 评估标准（Evaluation Rubrics）

Evaluator Agent 在验证所有 `acceptance_criteria` 之后，根据项目类型追加以下检查项。

## REST API 项目

- success path 返回正确 HTTP 状态码（200/201/204）
- error path 返回明确错误消息，不暴露 stack trace 或内部实现细节
- 未授权请求返回 401，权限不足返回 403
- 必填字段缺失返回 400，响应体包含具体字段名
- 边界值测试：空字符串、超大数值、特殊字符（`<>'"&`）

## CLI 项目

- `--help` 输出可读且准确，包含所有选项
- 错误时退出码非零（exit 1），成功时为 0
- 错误信息输出到 stderr，正常输出到 stdout
- 无效参数有明确错误提示，不 panic

## 前端 / UI 项目（配合 Playwright MCP）

- 关键用户路径可完成，不需要猜测或摸索
- 表单验证有即时反馈，错误信息在字段附近展示
- 错误状态有可读提示（不是空白页或 500 页面）
- 颜色、字体、间距有统一体系（不是随机堆砌的默认样式）
- 移动端 viewport（375px）不溢出，关键内容可见

## 后台任务 / Worker

- 幂等性：重复触发不产生副作用（多次执行同一任务结果一致）
- 失败有可重试机制，或记录明确的错误日志
- 不吞异常（at least log it）

## 数据库 / 存储层

- 写操作有事务保护，失败时数据一致
- 查询不存在的记录返回 null / 404，不抛出未处理异常
- 有适当索引（查询条件对应的字段）
- 无裸 SQL 字符串拼接（防 SQL 注入）

## 通用（所有项目追加）

- 新增代码中无 TODO/FIXME 注释（除非在 acceptance_criteria 中明确标注为已知限制）
- 新增功能有对应测试，且测试实际运行通过（非 skip/pending）
- 无硬编码密钥、token、密码
- 无明显的命令注入、SQL 注入、XSS 向量
